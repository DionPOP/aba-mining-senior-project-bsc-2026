Main Graph (Preferred)

Step by step: how to read this graph
  1. Top node: Read the gray main claim first (`good_claim` or `bad_claim`). This is the final conclusion being tested.
  2. Middle layer: Read incoming reasons around the claim. Green supports it, red challenges it. Dashed border means assumption (no direct evidence yet).
  3. Bottom layer: These are real phrases from customer reviews. They provide the evidence that feeds into higher-level reasons.
  4. Arrows and colors: Follow arrow direction to see influence flow. Black line = connection/support, red line = attack/counterargument.
  5. Badge number: The number on a node is evidence count. Higher number = mentioned by more reviews.
